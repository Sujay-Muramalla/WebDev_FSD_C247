The Big Picture: AI Meets Software Development
Imagine you ask ChatGPT to write a login system for your app — and within
seconds, it creates working code that connects to a database and authenticates
users.
Sounds amazing, right? But what if that same code accidentally allows anyone to
bypass login?
That’s the new reality of AI-generated code — fast, powerful, but often insecure.
This session explores how Generative AI and Agentic AI Systems are
transforming the Software Development Lifecycle (SDLC) — from design to
deployment — and what new risks come with that speed.
What Is SDLC (Software Development Lifecycle)?
The SDLC is the process developers follow to build secure, reliable software.
It includes stages like:
1. Planning – What do we need to build?
2. Design – How should it work?
3. Development – Writing the code.
4. Testing – Checking for bugs or vulnerabilities.
5. Deployment – Releasing it safely.
6. Maintenance – Keeping it updated and secure.

Generative AI in Development
Generative AI models like GitHub Copilot, ChatGPT, or Replit Ghostwriter can:
● Write entire code functions automatically.
● Suggest API integrations.
● Generate backend scripts or frontend pages.
● Even write Docker, YAML, or configuration files.
While this reduces manual work, it also introduces a hidden question:
“Did the AI just write something unsafe?”
The Problem: Speed Without Supervision
AI doesn’t understand intent or context — it predicts what
looks right based on
patterns in its training data.
That means if insecure code was common in its dataset, the AI might reproduce it
confidently.
Example:
You ask an AI to generate a login module. It writes this:
if username == "admin":
bypass_auth = True
It works, but it also gives full access to anyone who types “admin.”
This is why AI-assisted development requires security awareness at every stage.

Common AI-Generated Vulnerabilities1. Insecure APIs
APIs (Application Programming Interfaces) connect one system to another.
AI-generated APIs often:
● Lack authentication (anyone can access).
● Skip input validation (open to injection attacks).
● Forget rate limits (exposed to brute force attacks).
Analogy:
It’s like building a beautiful house — but leaving the front door wide open.
2. Secrets Exposure
A “secret” in development terms means anything private — passwords, API keys,
tokens, or encryption keys.
Sometimes AI-generated code includes these directly in the codebase, such as:
API_KEY = "mysecretkey123"
That’s like writing your ATM PIN on the card itself.
Once this code gets uploaded to GitHub, it’s automatically scanned and may leak
across repositories — leading to major breaches.
3. Configuration Mistakes
AI tools can generate YAML or ENV configuration files that expose sensitive data or
set permissions too broadly.
Example:
access_control: "allow_all"

4. Data Validation Gaps
When AI creates forms, login systems, or API endpoints, it may skip sanitizing

Without checks, attackers can inject malicious commands — leading to SQL
Injection or Cross-Site Scripting (XSS).
Secure Code Review: Why It Still Matters
Even when AI writes code, humans must review it.
Secure code review means reading through the program to ensure:
● Inputs are validated.
● Sensitive data is protected.
● Authentication and authorization are in place.
● No unnecessary permissions exist.
Analogy:
AI is like an enthusiastic junior developer — fast, but prone to rookie mistakes.
You still need a senior engineer (or security reviewer) to approve the work.

How Hackers Exploit AI-Supported Pipelines
AI now contributes to CI/CD pipelines (Continuous Integration / Continuous
Deployment).
Attackers target them to:
● Inject malicious code via automated merges.
● Exploit insecure YAML or JSON config files.
● Leak secrets stored in pipelines or environment variables.
Example:
An AI tool accidentally commits credentials in .env.
A hacker scans GitHub for leaked keys — finds yours — and gains cloud access
within minutes.

Auto-Generated Frontend & Backend Risks
AI models can generate complex web apps instantly.
However, these may contain flaws such as:
● Missing CSRF tokens in forms.
● Weak CORS configurations (cross-origin attacks).
● Unsecured database queries.
Real-World Analogy:
It’s like hiring a construction robot to build a house — it finishes in a day, but
forgets to install the locks.
AI & Infrastructure-as-Code (IaC) Dangers
AI also helps generate infrastructure scripts (e.g., Terraform, Kubernetes YAML).
But these can easily misconfigure permissions — such as:
● Public S3 buckets.
● Admin IAM roles.
● Open network ports.
These mistakes can expose entire systems to external attacks before the product
even launches.
